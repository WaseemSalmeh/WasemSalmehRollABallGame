# WasemSalmehRollABallGame

I downloaded Unity and learned how to use GitHub more before checking the steps of "Roll A Ball Game" creation on unity website.

THE TUTORIAL:

Setting up the game

i made a new 3d empty project and followed the first steps and created a new scene and made sure the layout is set to default before creating the ground of the game, the player, then the lighting system. i learned how to add new objects to the scene and edit them to make them look more realistic. Finishing those first steps i moved on to the next part of the tutorial which is about how to move the player.

Moving the player

learned how to move the sphere (the player) and knew how to script movements also used an AI tool to add jumping for the obstacles im gonna create later and to make the game more dynamic.

Moving the camera
I scripted the camera to make it follow the player though i changed its location to be closer to the player to be more immersive

Setting up the Play Area
Built a small kind of village as i added some small details and hidden places or secret places hard (but possible) to reach for collectibles to make gathering them more challenging. I didn't want just some plain stuff so i researched for more free interesting assets from the asset store to make the place feel more alive and natural as much as i could. Some of them were missing textures or didn't work properly so i reduced the amount of the objectives and placed some simple ones and ramps for fun as i used AI again for the grass and flowers to make them appear in a random pattern to fill the ground space instead of leaving it just as a solid color and lifeless. As for the textures in general they might look not the best but at least it's still better than just colors.

Creating Collectibles, Detecting Collisions with Collectibles, Displaying Score and Text
I distributed 11 collectibles appropriately around the map and made them glowing using AI so they would be more appealing and attractive as i made sure they work properly with the texts and the counter.

More details to map and few tricks
Added an extra collectable object and textures for the walls, used AI to make a script for the small objects to make them dynamic and react to the player's touch, though they might be little glitchy. Also a script that makes the game restart each time the player falls out the map. Fixed the lighting and changed the time to be an afternoon vibe.

Overall
I might have skipped a step from the tutorial which is making an enemy as i faced a lot of problems with making the enemy's AI to follow the player as it always gets lost or doesn't follow the player properly even though i made sure of everything is set right, it might be a glitch from the engine or the scripts that i couldn't find. It worked the first time i made it but later on it broke somehow and it got me stressful so i decided to cancel the idea and leave it at this. I learned a good amount of things while making this game.
